community-数据库-建表

community mysql 2019/04/05

题记

目前再做一个社区网站，以segmentfault思否平台为基础，逐渐完善其功能。本篇文章主要简述该项目的数据库设计。点击获取项目源码

基本规则

1、ID默认256字符
2、存在默认值或者范围需先定义，减少前后端验证代码量
3、采用小写英文字符和下划线组合形式定义表名和字段名。
4、重要数据需要增加一个字段is_delete判断用户是否已经删除，如果删除则预先保留一周时间，一周后自动删除。一周内可以找回数据。（回收站功能）
5、主要字段名：

  • user_id 作者ID
  • id 表ID
  • is_delete 是否删除 （1 删除，0 不删除）

评论表

reply_top_ID 顶级ID，这个是最重要的ID。用于前端展示层级关系。第一层级为文章ID，第二层级为第一层级的用户ID

账号表

保存用户名密码。用于登陆切换账号

个人档案表

个人基本信息表保存的都是一些非必填的用户需要额外增加信息的字段。

文章表

一次性保存文章所有信息，该表可能非常大。后期再修改吧

专栏表

根据文章类型进行专栏分类，系统默认存在一个专栏。

笔记表

草稿表

保存文章、笔记的草稿、日志

收藏表

主要收藏文章、笔记信息。
一个作者对应一个总收藏夹。一个总收藏夹对应多个收藏夹。

结尾

由于前期主要进行前端工作，后端和数据库的设计一笔带过，也没有详细的设计方案，暂时使用吧，等待前端工作基本完成后，再开始重新设计。
