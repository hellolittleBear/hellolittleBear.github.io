父子组件间通信问题

父子组件通信主要有$emit、props、$ref三种方式。

  • $emit用于子组件向父组件传递信息的。父组件绑定props数据传递传递到子组件，自组件通过$emit方法传递信息给父组件。
  • $ref主要用于调用子组件的属性和方法的。绑定子组件，则可调用子组件方法和属性，绑定原生DOM，一般用于选择器作用。
  • props则用于子组件接收父组件传递的属性的。另外，子组件是可以修改父组件传递的数据的。引用类型属性修改不告警，基本类型数据，则会发出告警，开发中一般不建议修改，毕竟文档追求的是 0warning;0error 的。但是在某些情况下（封装组件时，比如elementui中tree组件），就必须要修改，别无他法。
    最后，尤雨溪本人不反对这种方式，可以作为父子组件双向通信的一种简易方案。

eventBus多次触发问题

兄弟组件传递信息时使用eventBus;

出现问题原因，组件生命周期不理解：
兄弟组件跳转时生命周期函数：A -> B
A:beforeCreate() -> A:created() -> A:beforeMounted() -> A:beoforeUpdate() -> A:update() -> B:beoforeCreate() -> B:created() -> B:beforeMounted() -> A:beoferDestory() -> A:destory() -> B:mounted()

所以

  • $emit只能写在beforeDestory()或者destory()内
  • $on只能写在beforeMounted()之前
  • $on必须手动清除（beforeMounted()之后），否则会多次触发。
