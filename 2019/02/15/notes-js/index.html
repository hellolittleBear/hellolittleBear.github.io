JS笔记总结

18:56:00 前端
this指向问题

  • 当函数作为对象的方法被调用时，this就会指向该对象。比如 call、apply的this指向。

  • 当函数作为全局普通函数时， this指向window。

  • 构造器被调用时， this指向这个返回的对象。

  • 如果是箭头函数的话，this看所在函数定义在那个对象下，如果是嵌套，this指向最近的对象。

如何改变this指向

  • es6 箭头函数，强行绑定该函数
  • 保存this。 let that = this;
  • 借助apply、call、bind
  • new实例化对象
闭包、原型链
  • 产生： 当函数作为返回值时、函数作为参数传入时。
  • 优点： 可以构造私有方法、访问函数内部变量。
  • 缺点： 增加内存消耗，使用不当很容易内存泄漏。如非必要，不要用闭包。
内存溢出、内存泄漏
  • 内存溢出一般是开发意外写了死循环。比如重载setter函数里出现赋值表达式。
  • 内存泄漏是js中非常正常的事情。我们要看情况而定，一般开发者尽量减少内存泄漏发生。
  • 闭包会产生多余空间开销，但是不一定会产生泄漏问题。DOM对象和js对象循环引用以前会产生内存泄漏，因为以前IE使用引用计数进行GC，现在采用标记清除GC，不会了。await 递归循环也会产生内存泄漏。

数组去重

  • new Set()
    undefined、NaN被认为是相同的（虽然NaN != NaN）。+0、-0被认为是相同的（es5之前不同）。

  • indexOf查找splice删除

  • sort排序lastIndexOf查找splice删除

  • 利用对象属性名不同去重，不能分辨字符串数字，因为所有的项都回转换成字符串

  • 利用includes判断数组是否包含该值进行添加，去重

  • 利用filter、hasOwnProperty去重

  • 利用indexOf查找、filter过滤去重自动删除undefined、NaN
    function unique(arr = []) {
    return arr.filter((item, index) => {
    return arr.indexOf(item) === index;
    })
    }

  • 利用Maps数据结构的key值不同去重，与对象属性名不同类似

  • 利用reduce逐步判断前后俩值
